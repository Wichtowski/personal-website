# Samo-naprawiający się Test Runner AI

We współczesnych, zwinnych środowiskach programistycznych interfejsy użytkownika zmieniają się nieustannie. Skutkuje to kruchymi testami automatycznymi, które psują się z powodu drobnych zmian w klasach CSS lub modyfikacji struktury DOM. Aby temu zaradzić, zaprojektowałem **Samo-naprawiający się Test Runner AI**.

## Główny Problem

Standardowe potoki CI/CD (QA) zawodzą, gdy programiści frontendowi zmieniają klasy przycisków, zagnieżdżenie drzewa DOM lub identyfikatory selektorów. Testy zatrzymują się, co wymaga od inżyniera QA ręcznego zbadania problemu, przepisania testu i wypchnięcia poprawki.

## Rozwiązanie

Zintegrowaliśmy pętlę zwrotną pomiędzy narzędziem **Playwright** a lokalną/bezserwerową instancją **API Ollama/Claude**.

### Jak to działa:

1. **Wykrywanie**: Playwright nie może znaleźć selektora i zgłasza błąd.
2. **Budowanie kontekstu**: Uruchamiacz testów pobiera **Zrzut DOM**, wadliwy **Selektor** oraz **Log błędu**.
3. **Naprawa AI**: Lekki model językowy analizuje strukturę DOM, znajduje najbardziej pasujący semantycznie element dla brakującego selektora i tymczasowo go podmienia w pamięci.
4. **Wykonanie**: Test jest ponawiany z nowym selektorem. W przypadku sukcesu, system generuje **sugestię Pull Request**, aby na stałe zaktualizować kod!

```typescript
// Przykład funkcji inteligentnego kliknięcia
import { Page } from "@playwright/test";
import { healSelector } from "./ai-healer";

export async function smartClick(page: Page, fallbackSelector: string) {
  try {
    await page.click(fallbackSelector, { timeout: 2000 });
  } catch (error) {
    console.warn(`Selektor [${fallbackSelector}] zawiódł. Aktywacja leczenia AI...`);
    const healed = await healSelector(page, fallbackSelector, error);
    await page.click(healed);
  }
}
```

## Kluczowe Osiągnięcia

- **82% redukcji** czasu poświęcanego na utrzymanie potoku testowego.
- Automatyczne poprawki kodu wypychane bezpośrednio do gałęzi Git do weryfikacji.
- Pełna opcja lokalnego przetwarzania w celu ochrony własności intelektualnej i kodu źródłowego.
